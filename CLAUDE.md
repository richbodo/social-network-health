# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

Two loosely-coupled things live here:

1. **The `socialnetwork.health` website** — a TanStack Start (React 19) SSR app deployed to Cloudflare Workers, scaffolded by Lovable.dev. Despite the full shadcn/ui component library being present, the live homepage is a **single hand-written static HTML file** served verbatim (see below).
2. **`drafts/` — SNH research workstream.** Independent of the web app: a threat catalog, a `references.md` bibliography, a knowledge-base build plan, and a `paper-resolver` Python tool. These are documents and tooling, not part of the deployed site.

## Commands

Package manager is **Bun** (`bun.lock`, `bunfig.toml`).

```bash
bun install          # install deps
bun dev              # vite dev server (local development)
bun run build        # production build (Cloudflare)
bun run build:dev    # build in development mode
bun run preview      # preview the production build
bun run lint         # eslint .
bun run format       # prettier --write .
```

There is **no test runner configured** — `bun test` will find nothing. Don't claim tests pass; there are none.

Deployment targets Cloudflare Workers via `wrangler.jsonc` (worker name `tanstack-start-app`, `nodejs_compat`). Use `wrangler` for deploy/tail.

`bunfig.toml` enforces a 24h supply-chain guard (`minimumReleaseAge`) — packages published in the last day are skipped. Adding a package to `minimumReleaseAgeExcludes` requires confirming with the user first.

## Web app architecture

**The homepage content is `public/socialnetwork-health.html`, not React.** `src/routes/index.tsx` reads that file with `?raw` and returns it as a raw HTML `Response`. To change what the site says or looks like, **edit `public/socialnetwork-health.html`** (self-contained HTML + inline CSS). The React route tree, shadcn/ui components under `src/components/ui/`, and hooks are mostly inert template scaffolding for the live page — don't go looking for the site content in JSX.

**SSR error-handling layer** (the non-obvious custom code, all in `src/`):
- `src/server.ts` — the Cloudflare Worker entry (`wrangler.jsonc` `main`). Wraps the TanStack handler and, critically, **normalizes h3-swallowed errors**: h3 turns in-handler throws into a generic 200/500 JSON body `{"unhandled":true,"message":"HTTPError"}` that `try/catch` never sees, so `normalizeCatastrophicSsrResponse` sniffs for that exact body and replaces it with a branded error page.
- `src/lib/error-capture.ts` — registers global `error`/`unhandledrejection` listeners to stash the real `Error` out-of-band (5s TTL), so `server.ts` can recover a real stack after h3 has discarded it.
- `src/start.ts` — TanStack Start request middleware that catches throws during rendering and returns the branded error page (re-throwing objects that already carry a `statusCode`).
- `src/lib/error-page.ts` — the branded 500 HTML string used by both of the above.

**`vite.config.ts` gotcha:** the config uses `@lovable.dev/vite-tanstack-config`, which already bundles tanstackStart, viteReact, tailwindcss, tsConfigPaths, the Cloudflare plugin, the `@` path alias, and more. **Do not add those Vite plugins manually** — duplicates break the build. Pass extra config through `defineConfig({ vite: { ... } })`. The config also redirects the server entry to `src/server.ts`.

**Generated / convention files:**
- `src/routeTree.gen.ts` is auto-generated by the TanStack router plugin — never hand-edit (it's also in `.prettierignore`).
- `@/*` path alias → `src/*`.
- shadcn/ui is configured "new-york" style, base color slate, lucide icons (`components.json`). Tailwind v4 (CSS-based config in `src/styles.css`).
- ESLint forbids importing `server-only` (Next.js convention) — use `*.server.ts` or `@tanstack/react-start/server-only` instead.

## drafts/ — SNH research

Not part of the web app. Notable pieces:
- `threat_catalog.md` — SNH remote-worker threat model (`.v1.1.bak.md` is a backup; don't edit the backup).
- `references.md` — the corpus bibliography.
- `knowledge-base-plan.md` — the plan to turn the reference corpus into a markdown-per-paper knowledge base with CI-derived artifacts (catalog, card JSON, MCP server). Read this before working on corpus tooling.
- `skill_experiment/unzipped/resolve.py` — the **paper-resolver**: resolves DOIs/titles/topic-searches to metadata + *legal* open-access full text via OpenAlex → Unpaywall → Semantic Scholar → Europe PMC → Crossref. State lives in one SQLite db that is both cache and corpus index.

Running the resolver (see `drafts/paper-resolver-usage.md` for the validated invocations):
- **Global flags (`--db`, `--download`, `--extract`, `--out`, `--json`) go BEFORE the subcommand** (`doi`/`batch`/`search`); subcommand args like `--limit` go after. Putting `--json` after the DOI errors.
- Always set `RESOLVER_EMAIL=richbodo@gmail.com` (required by Unpaywall; polite pool for OpenAlex).
- Needs network access (sandbox off for the call) and PyMuPDF for `--extract`.
- `drafts/paper-corpus/` (the SQLite db + downloaded PDFs) is **gitignored** — large generated artifacts, not committed. The rest of `drafts/` is committed.
- The usage notes hardcode `/Users/rsb/...` paths; this checkout is under `/Users/richbodo/...` — adjust paths accordingly.
