This is a skill claude.ai made for me to find academic papers

paper-resolver.skill is ready to install (the loose SKILL.md and resolve.py are there if you'd rather drop them straight into a skills directory or read the source first).
What it does, briefly: given a DOI, a title-search, or a file of DOIs, it walks OpenAlex → Unpaywall → Semantic Scholar → Europe PMC → Crossref, takes the richest value for each metadata field, and grabs the first legal full-text URL it finds. Everything lands in one SQLite file that doubles as cache and corpus index, so reruns are cheap and idempotent. Paywalled papers with no open version get marked metadata_only and queued in unresolved.jsonl for manual pickup rather than the tool trying anything cute. Optional --download/--extract flags build you a folder of PDFs and extracted text to feed downstream.
I validated the offline pieces here (DOI normalization, OpenAlex abstract reconstruction, the field-merge precedence, SQLite round-trip, CLI parsing) — those pass. I couldn't make live calls to the scholarly APIs from this sandbox since they're outside its allowlist, so the first thing to do on your end is a single real DOI with your email set, to confirm the network path:
bashexport RESOLVER_EMAIL=you@example.com
python scripts/resolve.py doi 10.1371/journal.pone.0173644 --json
One genuinely-OA PLOS DOI like that should come back fulltext with a PDF URL, which tells you Unpaywall and OpenAlex are both responding.
Two notes worth keeping in mind. The email isn't optional in practice — Unpaywall requires it and it puts OpenAlex in the faster polite pool, so leaving it out quietly tanks your hit rate. And CORE isn't wired in: its v3 API needs a registered key, so I left it out of the default path rather than have it fail silently. If you want it as a tier, say so and I'll add it behind the key — it's the largest aggregator of actual OA full text, so it's a reasonable addition once you've got a key.
If you want, the natural next step is to point it at a real reference list from your SNH work and see what the full-text-vs-metadata-only split actually looks like — that ratio is the real test of whether the OA layer covers your field well enough, and it'll tell us whether the locked remainder is small enough to handle manually or worth chasing an affiliation for.
